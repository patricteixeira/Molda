# Plano M2 — Gate 0 de mecânica OOXML

**Objetivo:** eliminar o risco central de PPTX/DOCX nativos antes de integrar
arquivos editáveis à API ou à interface.

**Estratégia:** `Brand IR + Layout Spec + Content Spec → tema derivado →
template-fill → validação → round-trip → preview`. PPTX e DOCX compartilham a
marca e o conteúdo, não o layout engine.

**Estado em 14/07/2026:** implementação, goldens, duas marcas reais, round-trip
LibreOffice e preview isolado concluídos. Falta somente o round-trip obrigatório
em PowerPoint Desktop para fechar o Gate 0.

## Entregas

1. Validador defensivo de pacote OOXML: ZIP seguro, XML estrito, ausência de
   macros/embeddings/relations externas e relações slide→layout.
2. Tema OOXML derivado em cópia, com color scheme e major/minor fonts do Brand
   IR; o template original permanece imutável.
3. Spike A: PPTX preserva master/layout, texto nativo, imagem substituível e
   tags semânticas redundantes.
4. Spike B: parser reencontra role, texto, fonte e cor após save; DOCX preserva
   estilos `Brand *` após round-trip.
5. Spike C: preview por LibreOffice isolável, cuja falha é diagnóstica e nunca
   corrompe o documento.
6. Golden tests de XML canonicalizado desde o primeiro commit do M2.
7. CLI mínima: `native-theme`, `native-pptx`, `native-docx`, `native-inspect` e
   `native-preview`.

## Critérios de saída

- suíte estrutural e goldens verdes;
- duas marcas reais geram PPTX e DOCX sem finding bloqueante;
- preview PPTX sem overflow;
- save real no PowerPoint Desktop e reinspeção via `native-inspect`;
- conversão LibreOffice executada na imagem `native-preview.Dockerfile` com
  rede desabilitada;
- decisão registrada em `GO-NO-GO.md`.

## Fora deste gate

- múltiplos slides por deck;
- fitting completo e linter pós-save;
- parser/linter/fixer de M3;
- editor embutido;
- gráficos, SmartArt, mídia e macros.
